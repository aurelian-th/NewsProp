# Phase 4: Simulation & Agentic Layer

## 📝 Overview
The core engine. Agents will interact on the graph. We inject a news item from Phase 2, and agents transition through the SEIZ states based on their attributes (Phase 3) and the news item's controversy score.

* **Status:** Done
* **Locked By:** Phase 2 (Needs news items) & Phase 3 (Needs the network)
* **Unlocks:** Phase 5 (Analysis)

## 🛠️ Technical Stack
* `mesa` (Agent-based modeling framework).

## ⚙️ The SEIZ Logic (Step Function)
At each `step()`, agents interact with neighbors.
* **$S$ (Susceptible):** Default state.
* **$E$ (Exposed):** Agent encounters infected neighbor. They check their `critical_thinking` vs the news `emotional_intensity`.
* **$I$ (Infected):** Agent believes it. They broadcast to neighbors based on `transmission_probability` ($\beta$).
* **$Z$ (Skeptic):** Agent rejects it. They stop the spread on their edges.

## ▶️ Run

From `NewsProp` root:

```bash
python run_phase45.py --runs 10 --ticks 80 --output-dir outputs/phase45
```

Default behavior:

- If `outputs/phase3_network/mesa_payload_phase3.json` exists, Phase 4 uses that fixed Phase 3 network (preferred).
- If it is missing, the pipeline falls back to synthetic BA generation.

Force fallback mode explicitly:

```bash
python run_phase45.py --force-synthetic-network
```

Use a custom Phase 3 payload path:

```bash
python run_phase45.py --phase3-payload outputs/phase3_network/mesa_payload_phase3.json
```
