# Phase 2: AI & NLP Layer

## Overview
Standardize all scraped data into one schema, translate multilingual text to English, compute emotional impact with VADER, generate embeddings, and produce statistical artifacts needed by Phase 4 and Phase 5.

* **Status:** In Progress
* **Locked By:** Phase 1 data files
* **Unlocks:** Phase 4 (Simulation) and Phase 5 (Analysis)

## Finalized Method Decision
Because the corpus is mixed Romanian + Russian + English, the active strategy is:
1. Detect language per article.
2. Translate to English.
3. Run VADER on translated text.
4. Use VADER compound score as `impact_score` in range `[-1, 1]`.

Implementation is in `phase2/pipeline.py`.

## Technical Stack
* `vaderSentiment` for sentiment and emotional impact scoring.
* `deep-translator` for translation to English.
* `langdetect` for language detection.
* `sentence-transformers` for optional embeddings (`all-MiniLM-L6-v2`).

## Metrics Produced
* `emotional_score`: VADER compound in `[-1, 1]`.
* `emotional_intensity`: `abs(emotional_score)`.
* `emotional_density`: share of strongly emotional sentences.
* `impact_score`: alias of `emotional_score` for simulation input.
* `reference_bin`: emotional reference bucket (`very_low` to `very_high`).

## Intervals, CI, Calibration, Comparisons
* **Confidence Intervals:** Bootstrap 95% CI for mean `emotional_score` globally, by source, and by language.
* **Empirical Calibration:** Estimate `P(high_engagement | emotional_bin)` using top-quartile engagement as the positive event.
* **Comparisons:** Fake vs real mean difference and Cohen's d effect size.

## Immediate Next Step
Use `impact_score`, `emotional_intensity`, and calibration output as inputs for SEIZ transition probabilities in Phase 4.
